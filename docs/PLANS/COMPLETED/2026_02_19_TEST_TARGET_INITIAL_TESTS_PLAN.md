## Code Sketches

### SwiftMockKit-Generated Mock Usage

No hand-written mock needed. The `@Mockable` macro on `SessionLoading` protocol (in the main target) auto-generates `SessionLoadingMock` with:

```swift
// Auto-generated by @Mockable — DO NOT write manually
class SessionLoadingMock: SessionLoading {
    var discoverProjectsCallsCount = 0
    var discoverProjectsCalled: Bool { discoverProjectsCallsCount > 0 }
    var discoverProjectsThrowableError: (any Error)?
    var discoverProjectsReturnValue: [Project]!
    var discoverProjectsClosure: (() throws -> [Project])?

    func discoverProjects() throws -> [Project] {
        discoverProjectsCallsCount += 1
        if let error = discoverProjectsThrowableError { throw error }
        return try discoverProjectsClosure.map { try $0() } ?? discoverProjectsReturnValue
    }
}
```

**Test usage pattern:**
```swift
@Test func mockLoader_tracksCallCount() throws {
    let mock = SessionLoadingMock()
    mock.discoverProjectsReturnValue = [
        Project(id: "test", name: "test", path: "/test", sessions: [])
    ]

    let projects = try mock.discoverProjects()
    #expect(mock.discoverProjectsCallsCount == 1)
    #expect(projects.count == 1)
}

@Test func mockLoader_throwsConfiguredError() {
    let mock = SessionLoadingMock()
    mock.discoverProjectsThrowableError = NSError(domain: "test", code: 1)

    #expect(throws: (any Error).self) {
        try mock.discoverProjects()
    }
}
```

## Notes

- Swift Testing uses `@Test` attribute instead of `func test*()` prefix
- Use `#expect` for assertions, `#require` for preconditions that should fail the test
- `@Suite` groups related tests with a display name
- Tests run in parallel by default with Swift Testing
- The `@testable import Lumno` gives access to `internal` types
- SwiftMockKit's `@Mockable` macro generates mocks at compile time — no manual mock files needed
- Generated mock naming convention: `{ProtocolName}Mock` (e.g., `SessionLoading` → `SessionLoadingMock`)
- Mock features: call counting (`.CallsCount`), call verification (`.Called`), return value config (`.ReturnValue`), error injection (`.ThrowableError`), closure override (`.Closure`)
